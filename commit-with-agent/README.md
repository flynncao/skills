# AI 提交协作者 / ai-commit-co-author

---

## 简介

这个 Skill 用来在现有提交流程不变的前提下，把当前 AI 助手自动加入 Git 提交的协作者信息中。

**核心价值**

- 保留你已经在用的 `/commit` 或默认提交流程，只调整最终提交信息。
- 自动为提交补上 AI 协作者署名，方便团队追踪提交来源。
- 支持手动指定协作者来源，例如强制写入 Codex、Claude Code，避免依赖运行时识别。

**适用对象**

- `💻` 日常使用 AI 辅助开发的工程师 — 提交记录更清晰，协作归因更统一。
- `👥` 需要审计或复盘提交来源的团队 — 能直接在 Git 历史里看到 AI 协作者信息。

---

## 功能特性

### 核心功能

- 在生成提交信息后，自动追加对应的 `Co-authored-by` 协作者尾注。
- 支持识别当前运行中的 AI 助手，并写入匹配的协作者信息。
- 支持显式指定一个或多个协作者，例如 `claude`、`codex`，并按给定顺序写入。
- 自动去重，避免重复追加同一个协作者邮箱。
- 保留原有提交标题和正文，不改动原本的提交流程、暂存内容或后续推送行为。

---

## 使用指南

直接用自然语言描述你的提交需求，或在调用时补充希望写入的 AI 协作者。

### 常用说法速查

| 意图 | 示例话术 | 效果 |
| --- | --- | --- |
| 使用当前 AI 自动署名 | `提交这次改动，并带上当前 AI 协作者` | 按当前运行的 AI 自动追加对应协作者信息 |
| 强制指定 Claude Code | `$commit-with-agent claude` | 直接写入 Claude Code 的协作者尾注 |
| 强制指定多个协作者 | `$commit-with-agent claude,codex` | 按顺序写入多个协作者尾注 |
| 沿用原有提交流程 | `按正常 commit 流程提交，但补上 AI 协作者` | 保持原有提交方式，只修改最终提交信息 |

### 输出示例

提交完成后的提交信息会保留原始主题与正文，并在结尾追加协作者尾注，例如：

```text
feat: refine release script

Improve validation and error handling for release tags.

Co-authored-by: Codex <codex@openai.com>
```

## 使用场景

| 场景 | 角色 | 示例问法 | 收益 |
| --- | --- | --- | --- |
| AI 辅助编码提交 | 开发者 | `把这次修改提交，并带上当前 AI 协作者` | 提交历史能准确反映 AI 参与情况 |
| 指定协作者归因 | 开发者 | `$commit-with-agent codex` | 不依赖自动识别，直接控制署名结果 |
| 多助手联合产出 | 团队成员 | `$commit-with-agent claude,codex` | 一次提交中保留多个 AI 协作者记录 |
| 保持现有流程 | 维护者 | `继续用原来的 commit 方式，只补 co-author` | 无需改 Git 配置或钩子，接入成本低 |
